## Context

`ActionId` and `KeyAction` were closed literal union types. Plugins needed to register custom action IDs and key actions, but passing a plugin-defined string caused TypeScript errors. Widening the type to `string` would lose autocomplete for the 258+ built-in IDs.

## Problem

Plugins could not register custom action IDs or key actions without TypeScript errors. The `KeybindingService` also had a hard validation guard that rejected unknown key actions, breaking plugin-registered default keybindings. The keybinding profile importer silently failed on legacy profiles with empty-string action IDs.

## Root cause

Closed literal union types (`type ActionId = 'a' | 'b' | 'c'`) reject any string not in the union. Three separate issues compounded: the type definition, the runtime validation guard, and the import filter.

## Lesson

Use the `BuiltIn... | (string & {})` open union pattern to allow arbitrary strings while preserving IDE autocomplete for built-in values:

```ts
// Before — closed, rejects plugin IDs
type ActionId = 'terminal:new' | 'git:commit' | /* ... 256 more */;

// After — open, autocomplete still works for built-ins
export type BuiltInActionId = 'terminal:new' | 'git:commit' | /* ... */;
export type ActionId = BuiltInActionId | (string & {});
```

The `(string & {})` intersection is necessary — plain `string` collapses the union and loses autocomplete. TypeScript treats `string & {}` as a distinct type that widens to `string` at runtime but keeps the literal suggestions.

Also relax any runtime validation that checks against a closed set of known values — plugin-registered values are unknown at build time, so the guard must accept any string and only reject structurally invalid entries (empty strings, wrong format).

## Applies when

- Any extension point where a closed literal union needs to accept third-party string values
- Plugin system IDs, action IDs, key actions, panel kinds, agent registry keys
- The `PanelKind` type already uses this pattern as the existing reference implementation
