## Context

Agent terminals (Claude, Codex, Gemini) use full-screen TUIs. When the user dragged a panel divider or toggled the sidebar, the ActivityMonitor would flip the agent from idle to working because the TUI repainted.

## Problem

Calling `ptyProcess.resize()` sends SIGWINCH to the child, which makes any TUI program redraw its entire interface. That redraw produces PTY output bytes that the activity heuristics (output volume, pattern matches) interpret as the agent doing real work. The agent state transitions to "working" with no user interaction.

## Root cause

Output-based activity detection cannot distinguish between bytes caused by the agent doing work and bytes caused by the terminal redrawing itself in response to a resize. Any layout change can produce false positives.

## Lesson

Whenever the app performs an action that induces terminal output as a side effect (resize, SIGWINCH, clear, cursor queries, theme toggles), open a short suppression window on the activity monitor so output-based heuristics are ignored but busy-state timers are still extended. The pattern in ActivityMonitor is `resizeSuppressUntil = Date.now() + suppressionMs`; new side-effectful operations should reuse it or mirror it. Input-based signals (Enter key) should remain un-suppressed so legitimate user-initiated transitions still fire.

## Applies when

Adding any feature that resizes terminals, forces repaints, sends control sequences, or otherwise writes to the PTY outside of user keystrokes. Also when debugging "agent flips to working for no reason" after layout changes, theme switches, or focus changes.
