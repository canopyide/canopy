## Context

Canopy publishes a universal (x64 + arm64) macOS build via electron-builder. `node-pty` has two native binaries — `pty.node` and `spawn-helper` — that are rebuilt for Electron by `@electron/rebuild`.

## Problem

Universal builds crashed at the merge step with `lipo: fat file already has ARM64 architecture`.

## Root cause

`electron-rebuild` on an Apple Silicon host compiles native modules as **already-universal** Mach-O binaries (both architectures in one file). `@electron/universal` then tries to merge the x64-build copy with the arm64-build copy of the same file — both of which are already universal — and `lipo` refuses to combine two fat files that contain the same architecture. The fix is to tell `@electron/universal` to take only one copy instead of attempting to merge.

## Lesson

When a native module ships an already-universal Mach-O binary, declare it in `build.mac.x64ArchFiles` (glob) in `package.json` so electron-builder copies the x64-slice version verbatim instead of feeding both slices to `lipo`. For node-pty use `Contents/Resources/app.asar.unpacked/node_modules/node-pty/build/Release/**` to cover both `pty.node` and `spawn-helper`.

## Applies when

Adding any native module to the project, bumping node-pty, or diagnosing any electron-builder universal-mac build failure that mentions `lipo` or "already has ARM64 architecture".
