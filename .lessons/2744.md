## Context

The hybrid input bar's `queueSend()` deferred submission via `requestAnimationFrame` (ostensibly to batch and avoid reading mid-flush state). Submit sites throughout the component all routed through this deferred path.

## Problem

Paste-and-submit flows across two terminals silently dropped input. Copy from left pane, paste into right pane, add a comment, press Enter — sometimes the input vanished without ever reaching the agent. Reproducible but timing-sensitive.

## Root cause

The rAF deferral opened a ~16ms window between "user pressed Enter" and "read editor state + send". During that window, focus changes (blur events from overlapping paste/focus flows, or from the paste dialog losing focus) fired their own blur handlers which reset or cleared the pending send state. The submit callback, when it finally ran, read already-cleared state and sent nothing — no error, no user feedback.

CodeMirror 6 IME composition added a subtlety: on `compositionend`, the committed text is written to state in the same tick, but reading immediately before the tick ends yields stale text. This is why `setTimeout(0)` (macrotask) works for composition end but rAF does not — rAF runs inside the current frame's microtask/render pass, before composition commit lands.

## Lesson

Synchronous submit paths should stay synchronous. Do not defer Enter-to-send through `requestAnimationFrame` — it creates a window for focus-change races to cancel the submission. Replace rAF with a direct synchronous call; batch on the sending side if needed, not on the trigger side.

For CodeMirror 6 `compositionend`, defer with `setTimeout(0)` (macrotask) so the committed text has landed before you read it. `rAF` fires before the composition text is committed.

If a submit path ever appears to "drop" input silently, check for:

1. rAF/setTimeout deferral between keydown and state read
2. Blur handlers that mutate pending-send state
3. Focus policy that can lose focus between trigger and execution

## Applies when

Adding async deferral to any submit/commit path. Debugging "sometimes" dropped input in CodeMirror/input components. Handling IME composition events. Reviewing paste flows that move focus between panels.
