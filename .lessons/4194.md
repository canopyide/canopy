## Context

`PtyPool.getFilteredEnv()` unconditionally set `LANG=en_US.UTF-8` and `LC_ALL=en_US.UTF-8` on every spawned terminal. The code comment said "TUI reliability: ensure rich terminal capabilities for Claude/Gemini CLIs."

## Problem

Non-English users (Japanese, German, Chinese, etc.) had all terminal output forced to English. On minimal Linux installs that don't have `en_US.UTF-8` generated, a `setlocale` warning appeared in every terminal. Setting `LC_ALL` is particularly aggressive — it overrides all individual `LC_*` variables (collation, monetary, time, etc.).

## Root cause

The original intent was avoiding the `C`/`POSIX` locale (which breaks UTF-8 TUI output), but `TERM=xterm-256color` and `COLORTERM=truecolor` already handle terminal capabilities. `LANG`/`LC_ALL` control language and encoding, not terminal rendering. The fix only needs to ensure UTF-8 encoding, not force English.

## Lesson

When setting terminal environment variables for TUI compatibility, only override `LANG` if no UTF-8 locale is already present — never unconditionally set it. Never set `LC_ALL` as a catchall; it suppresses every individual `LC_*` variable. The correct pattern:

```typescript
if (!filtered.LANG || !filtered.LANG.includes("UTF-8")) {
  filtered.LANG = "en_US.UTF-8";
}
// Never set LC_ALL
```

The extracted `ensureUtf8Locale()` helper in `EnvironmentFilter.ts` is the canonical implementation.

## Applies when

Spawning PTY processes in Electron, writing environment filters for terminal emulators, or any code that touches `LANG`/`LC_ALL` for "safety."
