## Context

Canopy ships a `.deb` to Ubuntu users and wants to work on Wayland sessions (GNOME 42+, KDE 6) at native scaling.

## Problem

On Ubuntu 24.04+ Canopy crashed immediately on launch with exit code 132. On Wayland sessions text was blurry at fractional scaling and IME (CJK input) was broken.

## Root cause

Two independent Linux platform traps. (1) Ubuntu 24.04 restricts unprivileged user namespaces by default as an AppArmor confinement policy. Chromium's sandbox requires `userns` to work, and without it Electron's zygote segfaults on spawn. The user-facing workaround is `sysctl kernel.apparmor_restrict_unprivileged_userns=0` but that is not acceptable for an app installer. (2) Electron defaults to X11 even under a Wayland session — it runs through XWayland, which fixes the window to integer scaling and cannot forward IME events to the compositor. The Ozone platform must be selected explicitly before `app.whenReady()`.

## Lesson

Ship an AppArmor profile at `build/linux/canopy.apparmor` granting `userns` and install it via the `.deb` post-install script (`build.deb.afterInstall` in `package.json`). Also set `chrome-sandbox` SUID as a fallback. For Wayland: before `app.whenReady()`, when `process.platform === "linux"`, set `ozone-platform-hint=auto`, and conditionally `enable-features=WaylandWindowDecorations` plus `enable-wayland-ime`. Guard all flag additions behind the Linux check so macOS and Windows are untouched.

## Applies when

Packaging Electron for Debian/Ubuntu, or any user report of immediate-crash on Ubuntu 24.04+, blurry text at fractional scaling on Linux, or broken IME input on Linux.
