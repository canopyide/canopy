## Context

Offscreen terminal containers hold xterm instances for panels that are docked or hidden. They must be invisible and non-interactive, but xterm still needs to initialize with valid dimensions. The containers applied three CSS properties to achieve this.

## Problem

Both offscreen containers (`TerminalOffscreenManager` and `DockPanelOffscreenContainer`) used a triple-redundancy pattern: `content-visibility: hidden` + `left: -20000px` + `pointer-events: none`. The `content-visibility: hidden` property alone already handles rendering suppression, layout skipping, hit-testing removal, and accessibility tree exclusion. The `-20000px` offset forced the browser to compute positioning on every style recalculation even though content was already hidden — wasted layout work.

`TerminalResizeController` used `rect.left < -10000` as a visibility gate before calling `fit()`. This heuristic only worked because of the `-20000px` offset, not because it measured actual visibility. With the offset removed, this check would always pass even for genuinely invisible elements.

## Root cause

The CSS triple-redundancy likely originated from defensive coding before `content-visibility` was widely supported. Once `content-visibility: hidden` was added, the other properties became redundant but were never removed. The resize controller's visibility heuristic was written to match the specific offset value rather than using a proper visibility API.

## Lesson

`content-visibility: hidden` is sufficient for offscreen containers — it suppresses rendering, layout, hit-testing, and the accessibility tree for all descendants. Do not combine it with `left: -20000px` (forces wasted positioning computation) or `pointer-events: none` (redundant). Use `element.checkVisibility()` as the proper DOM API for visibility gating in resize controllers, not a `rect.left < -10000` positional heuristic.

## Applies when

Creating or modifying offscreen terminal containers, or any code that conditionally calls `fit()` based on whether a terminal is currently visible.
