## Context

Deleting a git worktree was throwing `fatal: '<path>' is a main working tree` — even when the path appeared to be a regular linked worktree. The error blocked users who had a worktree that was already externally deleted or in an unusual state.

## Problem

The `isMainWorktree()` helper used `isBareRepository` as its heuristic: a bare repo indicates this is the main working tree. This fails for non-bare repos (the common case) and also for edge cases where `git rev-parse --is-bare-repository` returns unexpected output. The result was a false negative — the helper said "not main worktree" when it actually was, and `git worktree remove` was called on it, producing the fatal error.

## Root cause

The correct way to identify the main worktree is via `git worktree list`: Git always lists the main worktree **first** in its output. The previous heuristic (bare repo detection) is unreliable because non-bare repos are the norm and `isBareRepository` would always return false for them, even if the path is the main worktree.

```ts
// Correct approach: compare path to the first entry from `git worktree list`
async isMainWorktree(worktreePath: string): Promise<boolean> {
  const result = await git.raw(['worktree', 'list', '--porcelain']);
  const firstWorktreePath = parseFirstWorktreePath(result);
  return path.resolve(worktreePath) === path.resolve(firstWorktreePath);
}
```

## Lesson

To detect the main git worktree, use `git worktree list` and check if the candidate path matches the **first entry** — that is how Git itself defines the main worktree. Do not rely on bare repository detection (`isBareRepository`) as a proxy, since most projects are non-bare and this will produce false negatives. Always normalize paths with `path.resolve()` before comparing to handle trailing slashes and relative references.

## Applies when

- Any code that needs to determine if a path is the main (primary) git worktree
- Worktree deletion guards that should block the main worktree
- Worktree management UIs that distinguish main vs linked worktrees
