## Context

Canopy's polling intervals, WebGL context budget, hibernation thresholds, and git concurrency were hardcoded. On memory-constrained machines with many open worktrees, these fixed values caused excessive memory use and event loop lag.

## Problem

Single fixed configuration meant optimal values for a powerful machine were too aggressive for a battery-constrained laptop with limited RAM. No dynamic adaptation to actual system conditions.

## Root cause

Parameters baked into module-level constants rather than driven by observed system state.

## Lesson

- **Central coordinator, process-local consumers**: `ResourceProfileService` in the main process evaluates conditions (memory pressure, event loop lag, battery state, worktree count) and broadcasts profile changes via IPC to each process (pty-host, workspace-host, renderer). Each process adjusts its own parameters without internal restructuring — the profile is a config object, not imperative commands.
- **Hysteresis prevents oscillation**: require 30s of sustained pressure before downgrading (Performance → Balanced → Efficiency) and 60s of relief before upgrading. Without hysteresis, workload spikes cause rapid thrashing between profiles.
- **Balanced profile must equal previous hardcoded defaults**: ensures no behavioral regression when the system first ships. Unit tests verify this equivalence explicitly.
- **Three-tier profiling is sufficient**: Performance / Balanced / Efficiency covers the useful range without over-engineering. Continuous numeric tuning creates unpredictable behavior; discrete named profiles are easier to reason about and test.
- **Profile-aware consumers use override injection**: `HibernationService` and `ProcessTreeCache` accept dynamic config overrides from the active profile. This avoids re-architecting internals while making them profile-aware.

## Applies when

Introducing adaptive behavior to any subsystem with configurable polling/scheduling/budget parameters. Always add hysteresis thresholds to prevent oscillation under fluctuating load.
