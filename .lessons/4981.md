## Context

Canopy uses Electron's `WebContentsView` per project, meaning the active window reference can change during project switches, project creation, or focus events. E2E tests interact with the app via Playwright's Electron driver.

## Problem

Nightly E2E tests were failing with `Target page, context or browser has been closed` errors. Affected both core and online test suites.

## Root cause

Test helpers (`project.ts`, `workflows.ts`) and individual test files captured the Electron `window` reference once at setup time (e.g. `const window = await app.firstWindow()`), then used it throughout the test. When project switches or WebContentsView focus changes invalidated the reference, all subsequent interactions threw.

## Lesson

Never cache the Electron `window` (or `page`) reference across lifecycle events in Playwright E2E tests. Instead, use a `getWindow()` helper that calls `app.firstWindow()` (or equivalent) fresh at the point of use. Project switches and WebContentsView transitions silently invalidate cached references without any warning — the error only surfaces on the next interaction.

Pattern:

```ts
// BAD — cached at setup
const window = await electronApp.firstWindow();

// GOOD — resolved fresh each time
async function getWindow() {
  return electronApp.firstWindow();
}
```

## Applies when

- Writing or modifying any E2E test that performs project switches or multi-window operations
- Any `beforeEach`/`beforeAll` that captures a window/page reference
- Adding new E2E helpers in `e2e/helpers/`
