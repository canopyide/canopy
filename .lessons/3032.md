## Context

The dev-server webview CSP is a tight localhost-only policy. WordPress Gutenberg (and similar block editors) load preview iframes from `blob:` URLs and reference images on arbitrary HTTPS origins.

## Problem

Loading Gutenberg admin pages produced `ERR_BLOCKED_BY_CSP` for the block editor iframe and broken images throughout the admin UI.

## Root cause

Two CSP blind spots: (1) no `frame-src` directive, so blob iframes fell through to `default-src` which did not allow `blob:`; (2) `img-src` did not include `https:`, blocking externally hosted images.

## Lesson

When adding a new CSP directive, also add `blob:` to `frame-src` (Gutenberg, PDF viewers, and many rich editors use blob URLs for embedded frames) and `https:` to `img-src` for any CMS-style content. When writing CSP tests, assert per-directive (e.g. check `script-src` independently of `default-src`) rather than matching the full concatenated CSP string — the latter breaks every time you touch an unrelated directive and hides real regressions.

## Applies when

Modifying `webviewCsp.ts`, debugging `ERR_BLOCKED_BY_CSP` on dev-preview or browser panels, or adding CSP tests.
