## Context

When an agent terminal is launched, the shell initializes (loading `.zshrc`, `.bashrc`, etc.), sentinel detection runs to confirm the shell is ready, and then the agent command is injected via `writeCommand()`. All of the shell init output and intermediate prompts remain visible in the terminal.

## Problem

Users saw shell startup noise (prompt lines, sourced script output, sentinel echo commands) before the agent output appeared. This was confusing — the terminal appeared to have random shell init text before the agent took over.

## Root cause / Design decision

The fix injects a clear-screen + clear-scrollback escape sequence (`\x1b[3J\x1b[H\x1b[2J`) immediately before writing the agent command. This fires after sentinel detection (or the timeout fallback), so the shell is confirmed ready before clearing. The non-agent code path is unaffected.

The escape sequence breakdown:

- `\x1b[3J` — clears the scrollback buffer
- `\x1b[H` — moves cursor to home position
- `\x1b[2J` — clears the visible screen

## Lesson

Use `\x1b[3J\x1b[H\x1b[2J` (not just `\x1b[2J`) to clear both the scrollback buffer and the visible screen before injecting agent commands. Using only `\x1b[2J` clears the screen but leaves the scrollback, so users can still scroll up and see the init noise. The sequence must be written to the PTY after shell readiness is confirmed to avoid clearing agent output.

## Applies when

- Adding new agent types that need a clean terminal on start
- Debugging shell init noise appearing in agent terminals
- Implementing "clear before launch" behavior in any PTY-based terminal feature
- Writing tests for the `writeCommand()` path in `lifecycle.ts`
