## Context

`WorkspaceService` uses a shared `PQueue` to schedule `git status` polls across all worktrees. Each `WorktreeMonitor` enqueues a poll task every 2s (active) or 10s (background). With 40 worktrees and `concurrency: 3`, each background worktree could wait 130+ seconds between updates.

## Problem

Two distinct bugs compounded each other:

1. **Duplicate enqueues**: `WorktreeMonitor.poll()` had no guard preventing a second call while a previous poll was still in-flight. Rapid polling intervals could queue the same monitor multiple times, wasting slots.

2. **Unsafe `pollQueue.clear()` on project switch**: `WorkspaceService.onProjectSwitch()` called `pollQueue.clear()` to drain the queue, but this silently orphaned in-flight promises without cancelling them — operations continued running but their results were discarded, potentially leaving monitors in inconsistent state.

3. **No priority**: the active worktree (user is looking at it) competed equally with 39 background worktrees for the 3 queue slots.

## Root cause

`PQueue.clear()` removes queued-but-not-started tasks but does not cancel tasks already running. Calling it during a project switch meant some monitors had their pending tasks dropped while in-flight tasks continued — no clean lifecycle boundary.

## Lesson

When using PQueue for long-lived recurring tasks:

- Add a **per-monitor dedup guard** (`_pendingPollPromise`) at the entry of `poll()` — return early if a poll is already in flight.
- Use **AbortController** signals passed to `pollQueue.add()` for clean cancellation. Abort the controller in `stop()` so queued tasks are dropped and in-flight tasks can check the signal.
- Avoid `pollQueue.clear()` from outside monitors — it doesn't abort in-flight work and creates a split between what the queue thinks is running and what actually is.
- Use **PQueue priority** to ensure the active/focused item always gets a slot ahead of background items.

## Applies when

Using PQueue for concurrent background polling where items have different urgency levels and a stop/restart lifecycle is required.
