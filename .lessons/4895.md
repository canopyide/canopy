## Context

`XtermAdapter` uses a `ResizeObserver` with a 50ms debounce before calling `performFit()`. `TerminalResizeController` caches the last known dimensions to suppress redundant resize IPC calls.

## Problem

Terminal panels rendered blank on Ubuntu AppImage. The terminal would open but show no content or prompt.

## Root cause

Two bugs combined: (1) Containers on Ubuntu AppImage report 0x0 dimensions at initial mount time. The debounced `ResizeObserver` waited 50ms, saw 0x0, and considered the fit done — so when real dimensions arrived later, no fit ran. (2) `TerminalResizeController` compared the new resize against `lastWidth/lastHeight` without first calling `proposeDimensions()` to get xterm's actual current dimensions. Stale cached values caused corrective resizes to be suppressed as "already at this size."

## Lesson

For the very first non-zero `ResizeObserver` observation, bypass the debounce and run `performFit()` synchronously. Track this with an `initialFitDoneRef` and reset it on unmount. In `TerminalResizeController`, always call `proposeDimensions()` to get the terminal's real current dimensions before comparing — never rely on cached `lastWidth/lastHeight` alone to decide whether to suppress a resize. Cache is for deduplication of known-good states, not for overriding what the terminal reports.

## Applies when

Any terminal resize or fit logic, especially initial render. Running on Linux AppImage or any environment where initial container dimensions may be 0x0.
