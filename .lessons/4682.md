## Context

Electron exposes Chromium feature flags via `app.commandLine.appendSwitch('enable-features', ...)` and `app.commandLine.appendSwitch('disable-features', ...)` before `app.ready`. These must be set before the app is ready.

## Problem

Default Chromium settings include features that are suboptimal for an IDE-style Electron app: BackForwardCache consumes memory for browser-style navigation that Canopy never uses, and memory reclamation/throttling features are not enabled by default.

## Lesson

Useful feature flags for Electron apps that manage multiple WebContentsViews:

**Enable:**

- `PartitionAllocMemoryReclaimer` — returns freed memory to the OS more aggressively (stable since Chromium 110+)
- `IntensiveWakeUpThrottling` — aggressively throttles background timers in backgrounded renderers

**Disable:**

- `BackForwardCache` — caches entire page states for browser back/forward navigation; not useful in single-page Electron apps and wastes memory

One flag that did NOT work: `V8FlushBytecodeOnContextDispose` — this is an internal V8 flag, not a Chromium feature flag. It must not be passed via `enable-features`.

Extract feature flag configuration into a testable helper function so it can be unit-tested without launching the full app.

## Applies when

- Electron apps managing multiple WebContentsViews or renderers
- Memory optimization at the Chromium level
- Apps that don't use browser-style back/forward navigation
