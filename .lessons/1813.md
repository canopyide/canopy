## Context

Toggling the sidecar open/closed in overlay mode left terminals blank until the user resized the window. In push mode (where the sidecar pushes the grid) terminals redrew correctly — only overlay mode was broken.

## Problem

The grid's batch-fit effect (which calls `FitAddon.fit()` on all visible terminals) ran off a ResizeObserver on the grid container. In overlay mode the sidecar floats above the grid, so the grid's own dimensions do not change — the ResizeObserver has nothing to observe and never fires. In push mode the grid physically shrinks, so the observer fires and fit runs.

## Root cause

ResizeObserver only reports physical layout changes. "Something visually changed" is not the same thing as "this element's box changed." Any layer that floats above content without reflowing it (overlays, modals, floating panels, toasts over terminals) is invisible to a ResizeObserver on the content.

## Lesson

When fitting xterm (or any canvas/size-sensitive component) inside a container that can be visually occluded without reflowing, ResizeObserver alone is not enough. The effect must also depend on the overlay/sidecar/modal state explicitly, so React re-runs fit when the overlay state changes even if the container's box did not. Subscribe to the relevant store (sidecarOpen, sidecarLayoutMode, etc.) via Zustand selector and add it to the effect dependencies. Same applies to dock/grid transitions, maximize toggles, and any absolute-positioned overlays.

## Applies when

Adding a new floating/overlay UI that can occlude terminals or other size-sensitive content. Debugging "terminal blank after X toggle" where X does not physically resize the container. Reviewing any xterm fit logic. Adding new panel kinds that need to refit on visibility changes.
