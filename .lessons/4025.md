## Context

`ErrorService` buffers errors in a `pendingQueue` (max 50 entries, drops oldest on overflow) when `canSendToRenderer()` returns false — i.e., when the window is destroyed, webContents is unavailable, or the window is still initializing. On renderer ready, `flushPendingErrors()` drains the queue. This buffering protects against startup-phase errors being silently lost.

## Problem

The buffering/flush path had no E2E coverage. If the flush broke, users would miss errors that occurred during initialization — with no indication that errors were dropped.

## Timing challenge

To test buffering, an error must be injected _before_ the renderer is ready but _after_ the Electron app starts. On fast machines, the window may become ready before an `electronApp.evaluate()` call completes, making naive injection unreliable.

## Solution

Add a `__TEST_FAULT_INJECT` global to `errorHandlers.ts` in dev/test mode only, accessible via `electronApp.evaluate()`. The global pushes directly into the `pendingQueue` synchronously, bypassing the `canSendToRenderer()` check. This ensures errors are in the queue before the renderer attaches its listeners regardless of machine speed.

```ts
// dev-only, in errorHandlers.ts
if (process.env.NODE_ENV !== "production") {
  (global as any).__TEST_FAULT_INJECT = (error: AppError) => {
    pendingQueue.push(error);
  };
}
```

## Lesson

When testing timing-sensitive IPC paths (errors that must arrive before the renderer is ready), naive `evaluate()` injection is unreliable because the window may already be ready. The pattern is: expose a `__TEST_FAULT_INJECT` global in dev mode that bypasses normal routing and writes directly into the internal buffer. This gives deterministic test control without restructuring production code.

The same pattern applies to any main-process queue/buffer that has a "not yet ready, buffer it" codepath — expose a dev-only inject point rather than trying to race the initialization sequence.

## Applies when

- E2E testing of "buffer until ready" patterns in Electron main process
- Testing startup-phase error handling, queued events, deferred IPC sends
- Any situation where the testability requires injecting state before a window/renderer lifecycle event completes
