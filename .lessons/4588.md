## Context

Canopy's dock uses Radix UI `Popover` to show terminal panels in a floating layer. When a terminal or CodeMirror editor inside the popover has focus, pressing Escape should be sent to xterm.js (to stop a running agent) rather than closing the popover.

## Problem

Radix UI's `DismissableLayer` intercepts `Escape` in the **capture phase** before xterm.js or CodeMirror ever see it. The result: pressing Escape inside a focused terminal closes the dock popover instead of stopping the agent.

## Root cause

Radix UI uses event capture (`addEventListener('keydown', handler, true)`) so it can dismiss before the event reaches inner elements. There is no built-in "if the focused element is a terminal, don't dismiss" option. The `onEscapeKeyDown` prop on `PopoverContent` provides an escape hatch — calling `event.preventDefault()` there suppresses the dismiss.

The subtlety: you must check **focus** at the time the keydown fires, not at mount time. The guard function checks whether `document.activeElement` is inside a `.xterm-helper-textarea` (xterm's focus container) or a `.cm-editor` (CodeMirror) before calling `preventDefault()`. Other elements inside the popover (e.g., panel header buttons) should still allow Escape to close the popover.

## Lesson

- When embedding focusable interactive elements (terminals, editors) inside Radix UI popovers/dialogs, use `onEscapeKeyDown` to conditionally block dismissal based on which element has focus.
- Radix's event capture happens before `keydown` reaches xterm.js — it is not enough to call `stopPropagation()` from within xterm.
- Extract the guard into a shared utility (`dockPopoverGuard.ts`) and unit test all focus scenarios (terminal focused, editor focused, button focused, nothing focused).

## Applies when

- Embedding xterm.js terminals or CodeMirror editors inside Radix UI Popover/Dialog/Sheet
- Any component where inner interactive widgets need to receive keyboard events that Radix would normally intercept
- Building dock or floating panel systems on top of Radix UI primitives
