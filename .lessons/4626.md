## Context

Help agent terminal panels needed to inline YouTube video embeds when the agent outputs a YouTube URL. Electron's Content Security Policy blocks arbitrary iframes, and the standard `youtube.com/embed/` domain includes tracking.

## Problem

Standard `youtube.com/embed/` iframes conflict with Electron's CSP headers. Additionally, the feature must be scoped to help panels only — embedding arbitrary URLs in coding agent or regular terminal panels is a security risk.

## Root cause

Electron applies strict CSP by default. `youtube.com` sets tracking cookies. Without a sandbox attribute, embedded frames have full script access.

## Lesson

- **Use `youtube-nocookie.com`** for YouTube embeds: same embed functionality, no tracking cookies, and a cleaner CSP story.
- **Add `sandbox` attributes to all iframes**: `sandbox="allow-scripts allow-same-origin"` prevents the embedded content from navigating the parent, accessing top-level frame, or triggering popups.
- **Gate the feature by panel type via an `isHelpPanel` prop**: threading a flag through `TerminalPane` → `useYouTubeEmbeds` prevents the hook from activating for non-help panels. Don't rely on runtime URL inspection alone — use explicit panel identity.
- **Cap embeds per panel** (3 max): prevents visual overload if an agent outputs many URLs. Implement via a Zustand store that tracks embeds per panel ID with an upper bound.
- **URL parsing must handle ANSI escape codes**: terminal output contains ANSI sequences. Strip them before URL matching — extract `stripAnsiCodes` as a shared utility.
- All standard YouTube URL formats need handling: `youtube.com/watch?v=`, `youtu.be/`, `youtube.com/embed/`.

## Applies when

Embedding any third-party iframe content in an Electron renderer. Always use privacy-respecting domains, sandbox attributes, and explicit feature gating by context.
