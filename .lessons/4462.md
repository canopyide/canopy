## Context

`TrashedPidTracker` tracks PIDs of PTY processes that were active during a crash. On restart it compares each tracked PID's creation time against the current process at that PID to determine if it's the same process (should be killed) or a recycled one (should be skipped).

## Problem

On Windows 11, `wmic` no longer exists. It was deprecated in Windows 10 21H1 (May 2021) and fully removed from Windows 11. The code called `execFileSync('wmic', ...)` to get a process creation time. This threw `ENOENT`, which was caught and returned `null`. `null` never matched any expected start time, so `verifyProcessStartTime()` always returned `false` — every orphaned PTY process was silently skipped and never cleaned up.

The failure was completely silent: no error surfaced to the user, the tracker appeared to work, but orphaned shell processes accumulated indefinitely across crash/restart cycles.

## Root cause

Silent error handling (`try/catch → return null`) masked a hard dependency on a system tool that no longer exists. `wmic` was removed from Windows 11 with no compatibility shim.

## Lesson

- **`wmic` is dead on Windows 11.** Replace all `wmic` usage with `Get-CimInstance Win32_Process` via `powershell.exe`. This is the same API, and `ProcessTreeCache.ts` already used it — verify parity within the codebase before reaching for `wmic`.
- **Silent null returns hide missing system tools.** If a cross-platform helper returns `null` on failure, make sure callers treat `null` as "skip this item" only when that's the intended semantic, not as a silent no-op that allows broken behaviour.
- **PowerShell output format differs from wmic**: PowerShell returns ISO-style `DateTime` strings; `wmic` returned `YYYYMMDDHHMMSS.mmmmm±ZZZ`. The timestamp parsing regex must be updated alongside the command change.
- When adding new Windows system calls, check `ProcessTreeCache.ts` first — it likely already has a working `Get-CimInstance` example.

## Applies when

Any code that calls `wmic` for Windows process metadata. All such callsites should be migrated to PowerShell `Get-CimInstance`.
