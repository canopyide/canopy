## Context

Codex and Gemini use Ratatui (Rust TUI library) to render their prompt UI. When Canopy switches projects or worktrees it reattaches terminals, triggering a post-wake "bounce" resize and a pre-warm resize path for offscreen agents.

## Problem

Switching projects/worktrees left Codex terminals visually corrupted — overlapping text, mis-positioned cursor, ghost lines.

## Root cause

Ratatui redraws on every `SIGWINCH`. When PTY resize and xterm.js resize happen on different frames, Ratatui sees two dimension changes back-to-back and the second redraw writes into an already-repainted buffer. Additionally, the "post-wake bounce" (resize-then-resize-back to force a fresh paint) is actively harmful to Ratatui which treats every resize as authoritative. Stale `settled` timers from a previous project could also fire a resize after detach. And injecting clear-screen via PTY stdin (rather than through xterm.js) was racing with Ratatui's own output stream.

## Lesson

For settled agents (Codex, Gemini, anything detected as Ratatui-based): (1) skip the post-wake resize bounce entirely — it corrupts Ratatui state; (2) defer the xterm.js resize so it fires atomically with the PTY resize in the same frame; (3) inject clear-screen via xterm.js, not by writing `\x1b[2J` to PTY stdin, because the TUI owns the PTY output stream and will overwrite you; (4) clear the settled timer on project-switch detach so no stale resize fires after the terminal is gone; (5) skip the duplicate prewarm resize for offscreen agents — `FitAddon.fit()` already handles initial sizing; (6) route all prewarm resizes through the settled-aware `sendPtyResize()` so they respect the Ratatui guards.

## Applies when

Touching resize paths in `TerminalResizeController`, `TerminalInstanceService`, `TerminalRegistryController`, or anywhere that sends a resize to an agent terminal running Codex, Gemini, or any Ratatui/ncurses full-screen TUI.
