## Context

Electron renders a blank white (or black) window during the period between `show` and the first React render completing. This is especially visible on first launch or after cache invalidation when V8 compilation takes longer.

## Problem

Users saw a blank window for a perceptible period at startup, giving the impression of a slow or broken app.

## Lesson

Inline a skeleton UI directly in `index.html` using CSS custom properties that match the app's theme tokens. The skeleton loads with zero JavaScript, appearing the instant Chromium paints the first frame.

Key implementation details:

1. Use CSS custom properties for colors so the skeleton respects both light and dark themes without JavaScript.
2. Add a pulse animation with `animation-fill-mode: forwards` so the skeleton is visible immediately (no 400ms fade-in delay).
3. Remove the skeleton after React's first render using `requestIdleCallback` with a safety timeout fallback (3s). This ensures the skeleton is cleared even if `requestIdleCallback` is not supported or never fires.
4. Make the removal idempotent — the element may already be gone if called twice.
5. Call the removal utility from `App.tsx` `useEffect` on mount.

The skeleton layout should approximate the real app structure (sidebar, toolbar, main content) to avoid a jarring layout shift when the real UI appears.

## Applies when

- Reducing perceived startup time for Electron apps
- Any BrowserWindow that shows a blank period before the React app renders
- Adding a new top-level window with a distinct layout
