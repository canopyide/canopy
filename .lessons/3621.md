## Context

Electron's main process had compile caching enabled via `enableCompileCache()` in `bootstrap.ts`, but the renderer process had neither V8 bytecode caching nor code cache enabled for the custom protocol scheme.

## Problem

The renderer re-parsed and re-compiled all JavaScript on every launch despite the main process having compile caching already configured. A two-line change was all that was needed.

## Root Cause

The main process and renderer are separate V8 contexts with separate caching configurations. Enabling caching in one does not affect the other.

## Lesson

Two separate settings are required for full renderer V8 caching in Electron:

1. `v8CacheOptions: "code"` in `BrowserWindow.webPreferences` — enables bytecode caching for the renderer's V8 context
2. `codeCache: true` in the custom protocol scheme privileges registered via `protocol.registerSchemesAsPrivileged()` — enables the code cache for resources served via that scheme

Both must be set. The main process `enableCompileCache()` only covers the main and utility process V8 contexts, not the renderer.

## Applies when

- Optimizing Electron app startup time
- Adding a new BrowserWindow or WebContentsView
- Registering a custom protocol scheme (`app://`, `file://` override, etc.)
