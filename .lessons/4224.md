## Context

The `postinstall` script previously ran `electron-rebuild -f -w node-pty,better-sqlite3`. The `-f` flag forces source compilation, ignoring any shipped prebuilt binaries.

## Problem

On Windows, compiling node-pty from source requires the MSVC "Spectre-mitigated libraries" VS component (`MSVC v143 - VS 2022 C++ x64/x86 Spectre-mitigated libs`), which is not included in the standard C++ workload. Developers installing VS 2022 Build Tools with the default C++ workload get a cryptic `MSB8040: Spectre-mitigated libraries are required` build error. Both `npm install` and `npm run package` fail. node-pty v1.2.0-beta.12 ships Windows prebuilts in `prebuilds/win32-x64/` that are completely bypassed by `-f`.

## Root cause

The `-f` (force) flag in `electron-rebuild` bypasses prebuild detection. The flag was likely added to ensure a clean rebuild after Electron version changes, but it prevents using prebuilds on Windows where source compilation has extra requirements. Switching to `@electron/rebuild` with `buildType: 'Release'` and `useElectronClang: false` allows the tool to detect and use node-pty's prebuilts on Windows, while still compiling from source on macOS/Linux where the build environment is reliably available.

A `beforeBuild.cjs` electron-builder hook is also needed to copy prebuilts into `build/Release/` during packaging, since electron-builder's packaging step doesn't run the same prebuild-resolution path.

## Lesson

Never use `electron-rebuild -f` in `postinstall` on a multi-platform project if any native module ships prebuilts. The `-f` flag is useful for CI cleanup but harmful as a default for developer onboarding. Prefer `@electron/rebuild` (the successor package) without force-rebuilding, so prebuilts are used when available and source compilation is a fallback. For packaging, add an explicit `beforeBuild` hook to copy prebuilts into the expected location.

## Applies when

- Modifying `postinstall` or rebuild scripts that compile native modules
- Adding new native module dependencies to the project
- Debugging Windows build failures with MSVC/node-gyp errors
- Updating node-pty or better-sqlite3 versions
