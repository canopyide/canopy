## Context

Terminal output data was flowing: pty-host → main process (IPC serialization) → renderer. With 30–40 concurrent terminals producing heavy AI agent output, the IPC path became the primary bottleneck, causing silent data drops when the queue filled.

A zero-copy SharedArrayBuffer ring buffer architecture (`SharedRingBuffer.ts`, `PacketFramer`, `PacketParser`) was already implemented but permanently dead because `PtyClient.getSharedBuffers()` always returned empty arrays. Root cause: Electron's `UtilityProcess` does not support SharedArrayBuffer transfer.

## Problem

Two problems interacted:

1. SharedArrayBuffer cannot be transferred to/from `UtilityProcess` in Electron — the entire high-throughput path was dead code
2. The IPC fallback still serialized every data frame through the main process, creating GC pressure and latency

## Root cause

The `UtilityProcess` constraint (no SAB support) means the SAB-based zero-copy architecture is fundamentally blocked in Electron. The correct alternative is `MessagePort` — which CAN be transferred to a `UtilityProcess` — enabling direct pty-host → renderer data flow without a main-process hop.

## Lesson

**Electron `UtilityProcess` cannot transfer SharedArrayBuffers** — this is a hard platform constraint, not a configuration issue. If you see a SAB-based ring buffer in `pty-host/` that's always returning empty arrays, it's permanently blocked.

The correct high-throughput path for `UtilityProcess` → renderer is `MessagePort`:

- Transfer a `MessagePort` to the UtilityProcess at spawn time
- Send `ArrayBuffer` frames over the port (these CAN be transferred)
- Implement watermark-based backpressure (`PortQueue` with high/low watermarks) to coordinate PTY pause/resume with port buffer state
- Keep the IPC fallback for terminals spawned before the port is established

When adding port backpressure, the `PtyPauseCoordinator` must support multiple concurrent pause reasons (port pressure + IPC pressure) so they don't interfere.

## Applies when

- Any high-throughput data path involving `UtilityProcess` — default to MessagePort, never SAB
- Debugging terminal data loss under heavy agent output — check port queue watermark thresholds
- Extending `PtyPauseCoordinator` — preserve multi-reason pause support
