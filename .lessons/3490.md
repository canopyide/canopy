## Context

Vite 8 ships with Rolldown as the bundler (replacing Rollup) and `@vitejs/plugin-react` v6 uses Oxc for transforms (replacing Babel). The migration required updating the code-splitting API and the console stripping mechanism.

## Problem

Two API changes broke the existing Vite 6 config:

1. `build.rollupOptions.output.manualChunks` is deprecated/removed in Vite 8 — Rolldown uses a different code-splitting API
2. `esbuild.pure` (used to strip `console.*` calls) is not available under Rolldown — it uses native `define`/`drop` mechanisms

## Root cause

Vite 8 / Rolldown is a significant architectural change — Rolldown has its own chunking API (`build.codeSplitting`) that differs from Rollup's `manualChunks`. Console stripping via `esbuild.pure` worked in Vite 6 because esbuild was the transformer; Rolldown has its own equivalent.

## Lesson

When migrating to Vite 8:

- Replace `rollupOptions.output.manualChunks` with `build.codeSplitting` using named groups:

  ```ts
  build: {
    codeSplitting: {
      strategy: 'custom',
      customSplitting: { vendor: [...matchers] }
    }
  }
  ```

- Replace `esbuild.pure: ['console.log', ...]` with Rolldown's native drop:

  ```ts
  define: { 'console.log': 'void 0' }
  // or use build.drop: ['console']
  ```

- The React plugin v6 uses Oxc — Babel-specific plugin options are removed
- Anchor regex patterns in code-splitting matchers to avoid false matches outside `node_modules`

Also remove `@vitejs/plugin-react`'s Babel dependency from `package.json` — it's no longer needed.

## Applies when

Upgrading from Vite 6 to Vite 8 in any Electron or web project using custom chunk splitting or console stripping.
