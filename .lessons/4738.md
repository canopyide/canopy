## Context

The codebase had 62 instances of Tailwind's `transition-all` class across hot-path components (dock tabs, panel headers, portal toolbar, button variants).

## Problem

`transition-all` tells Chromium to watch and interpolate every computed CSS property on the element. In Electron's Chromium renderer, this means the compositor must check all properties on every frame during any transition — even properties that never change on that element. On hot-path components like `DockedTerminalItem` (rendered once per open terminal) and `PanelHeader`, this adds up.

## Root cause

`transition-all` is a convenient default during development — it makes every hover/active state "just work". But it becomes a rendering tax in production, particularly in components that are rendered many times or that animate frequently.

## Lesson

Use specific transition properties instead of `transition-all`:

| When you need           | Use instead of transition-all                   |
| ----------------------- | ----------------------------------------------- |
| Color/bg/border changes | `transition-colors`                             |
| Width/height changes    | `transition-[width]` / `transition-[height]`    |
| Opacity changes         | `transition-opacity`                            |
| Transform changes       | `transition-transform`                          |
| Multiple specific props | `transition-[color,background-color,transform]` |

Tailwind v4's base `transition` class targets `color, background-color, border-color, text-decoration-color, fill, stroke, opacity, box-shadow, transform, filter, backdrop-filter` — use it as a safe default when you need several properties but not all.

The fix was a codebase-wide sweep of 40 files. Establish the correct pattern in `button.tsx` and other base components early so leaf components inherit the right default.

## Applies when

- Any component rendered more than once per screen (list items, tab items, toolbar buttons)
- Any component that has hover/active/focus transitions
- Code review: flag `transition-all` in hot-path components
