## Context

`XtermAdapter` uses a `ResizeObserver` plus a `visibilitychange` listener to trigger `performFit()` when a terminal panel becomes visible. On Windows, tab switching causes a rapid hiddenтЖТvisible transition.

## Problem

Terminal panels rendered blank on Windows when switching tabs. The panel appeared but showed no content.

## Root cause

Two separate races:

1. The `ResizeObserver` callback was gated on `document.visibilityState === "visible"`, so layout dimensions captured while a panel was briefly hidden during tab switching were discarded тАФ even though those dimensions were valid.
2. The `visibilitychange` listener was registered in `useEffect`, which runs after paint. On Windows, the hidden-to-visible transition could fire between render and the `useEffect` callback, meaning the event was missed entirely and `performFit()` never ran.

## Lesson

Use `useLayoutEffect` (not `useEffect`) for event listeners that must catch DOM transitions that could fire immediately after render. Always add a synchronous "did I already miss this event?" check after registering тАФ read the current state and call the handler if the transition already occurred. Do not gate `ResizeObserver` callbacks on `document.visibilityState`; panel dimensions can be valid even when the document is temporarily hidden during transitions.

## Applies when

Adding `visibilitychange` or other DOM transition listeners in React components where the event can fire in the gap between render and effect. Terminal resize / fit code on any OS, but especially Windows where transitions are faster.
