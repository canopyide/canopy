## Context

`TerminalPane` displays a badge showing how many commands are queued for each terminal. The badge count was derived from the command queue store on every render.

## Problem

The selector used `Array.filter()` to count commands for a specific terminal ID, scanning the entire queue on every render of every `TerminalPane`:

```ts
// O(N×M): N terminals × M queued commands, evaluated on every re-render
const count = useTerminalStore((s) => s.commandQueue.filter((c) => c.id === id).length);
```

With multiple terminal panes open and frequent queue updates, this created O(N×M) work that scaled poorly.

## Root cause

Derived count was computed at read-time instead of being maintained at write-time.

## Lesson

**Maintain derived counts at write-time in Zustand stores.** For any store value that is a filtered/reduced count of another collection, add a `countById: Record<string, number>` map and keep it in sync at the three mutation points (add, remove, clear):

```ts
// State
commandQueueCountById: Record<string, number> = {};

// In queueCommand():
state.commandQueueCountById[id] = (state.commandQueueCountById[id] ?? 0) + 1;

// In processQueue():
state.commandQueueCountById[id] = Math.max(0, (state.commandQueueCountById[id] ?? 1) - 1);

// In clearQueue():
delete state.commandQueueCountById[id];
```

The selector then becomes O(1) and returns a primitive (number), so `useShallow` is no longer needed:

```ts
const count = useTerminalStore((s) => s.commandQueueCountById[id] ?? 0);
```

## Applies when

Any Zustand selector that filters or reduces a collection to produce a per-ID count or aggregate. The rule: if a derived value requires scanning a collection, compute it at write-time instead.
