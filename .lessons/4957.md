## Context

`ScrollShadow` is a shared wrapper component used to add scroll shadows to content areas inside dialogs.

## Problem

The recipe editor dialog didn't scroll when content overflowed — the dialog grew to cover screen area or clipped content instead.

## Root cause

`ScrollShadow` used `h-full` which doesn't establish a scrollable container in flex contexts. In a flex column parent without explicit height constraints, `h-full` resolves to `auto` and the element expands with its content rather than scrolling.

## Lesson

In flex column layouts, `h-full` does not constrain height for scrolling purposes. Use `flex flex-col flex-1 min-h-0` instead:

- `flex-1` allows the element to take available space
- `min-h-0` overrides the default `min-height: auto` that prevents flex children from shrinking below their content size
- Without `min-h-0`, the flex child can overflow the parent regardless of `overflow-y: auto`

This is a Tailwind/CSS flexbox gotcha that affects any scrollable region inside a flex container.

## Applies when

- Building any scrollable content area inside a flex column container (dialogs, sidebars, panels)
- Using `ScrollShadow` or any wrapper with `overflow-y: auto`/`scroll`
- Replacing `h-full` in a flex context where scrolling isn't working
