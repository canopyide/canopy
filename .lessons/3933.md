## Context

Claude Code's working state is detected by matching spinner Unicode characters + status verb + "esc to interrupt" in PTY output. The character class and completion format in Canopy's config drifted from the actual CLI output.

## Problem

Three issues in Claude detection patterns:

1. **Wrong spinner character class.** The class `[✽✻✼✾⟡◇◆●○]` included 7 chars Claude doesn't use and was missing 4 it does: `·` (U+00B7), `✢` (U+2722), `✳` (U+2733), `∗` (U+2217). The `·` middle dot is the **first animation frame** — the most-displayed character. Missing it meant the primary pattern failed on the first frame of every spinner cycle, falling back to the secondary "esc to interrupt" pattern with potential latency.

2. **Hardcoded verb list breaks custom spinnerVerbs.** The fallback pattern listed specific verbs `(thinking|deliberating|working|reading|writing|searching|executing)`. Claude Code v2.1.23+ allows users to configure custom `spinnerVerbs` in `settings.json`. Custom verb users would never match this fallback.

3. **Outdated completion format.** The completion pattern matched a cost/token summary (`$0.XX (Xk tokens)`) but Claude Code now outputs `"Worked for Ns."` on completion.

## Root cause

Agent CLI output formats change with CLI releases. The spinner character class was likely sourced from an older version of `cli.mjs`. The verb list was written before the custom spinnerVerbs feature existed. The completion format changed silently in a Claude Code update.

## Lesson

Agent detection patterns must track the agent CLI version they were validated against. The spinner character class should be re-verified against the actual CLI source (`cli.mjs` or equivalent) when the agent CLI has a major version bump. Use structural matching (`\S+`) for verbs rather than hardcoded lists to survive custom configuration. Maintain two completion patterns: the current format plus the legacy format, since users may be on older CLI versions.

## Applies when

Investigating why Claude Code agents show incorrect working/idle state, or when updating `agentRegistry.ts` after a Claude Code version bump.
