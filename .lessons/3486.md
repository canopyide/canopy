## Context

Vite defaults to the `modules` preset for `build.target`, which targets `['es2019', 'edge88', 'firefox78', 'chrome87', 'safari14']`. Canopy's renderer only ever runs in Electron's bundled Chromium (version 144 in Electron 40, higher in Electron 41). Vite was emitting compatibility shims and polyfills for features Chromium 144 supports natively.

## Problem

The default Vite target produced larger bundles with unnecessary polyfills (e.g., for features like optional chaining, nullish coalescing, top-level await) that the actual runtime supports natively. Also, `modulePreload` polyfill was emitted despite native module preload support.

## Root cause

Vite doesn't know which Chromium version will run the output — it defaults to a conservative multi-browser target. The `modules` preset is appropriate for public web apps but overly conservative for Electron, where the exact Chromium version is known.

## Lesson

Always set `build.target` to the actual Chromium version bundled with the Electron version in use:

```ts
build: {
  target: 'chrome144', // match Electron 40's Chromium
  modulePreload: { polyfill: false }, // native support available
}
optimizeDeps: {
  esbuildOptions: { target: 'chrome144' } // keep dev/prod consistent
}
```

Check `electron/chromium-versions` or the Electron release notes for the exact Chromium version. Update `target` when upgrading Electron. The dev build (`optimizeDeps.esbuildOptions.target`) must match production to avoid behavior differences between dev and prod.

**Note:** As of Electron 41, the Chromium version is higher — verify and update `target` on each Electron major upgrade.

## Applies when

Any Electron app using Vite. The default `modules` target is always wrong for Electron — set it explicitly.
