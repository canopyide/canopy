## Context

Canopy detects agent working state by matching spinner characters, status verbs, and interrupt hints in PTY output. Pattern definitions live in `shared/config/agentRegistry.ts` and are mirrored in `electron/services/pty/AgentPatternDetector.ts`.

## Problem

Two agents had patterns that never matched real output:

**Cursor:** The completion pattern `⬢\s+(Thought|Read|Planned)\s` only covered 3 of 8 past-tense verbs. Missing: `Searched`, `Ran`, `Edited`, `Grepped`, `Listed`. When Cursor finished a search or edit, it stayed stuck in "working" state until the 6-second idle timeout fired. The `\s+` whitespace requirement also broke for output formats like `⬢Thought3s` where Cursor omits the space.

**OpenCode:** The braille spinner characters `[⠋⠙⠹⠸⠼⠴⠦⠧⠇⠏]` were copied verbatim from Gemini's detection config. OpenCode (Bubble Tea TUI) uses a completely different braille set: `⣾⣽⣻⢿⡿⣟⣯⣷`. None of the patterns ever matched. The interrupt hint also used `"Press again to interrupt"` but OpenCode outputs `"Press Esc again to interrupt."`.

## Root cause

Pattern authors copy-pasted spinner characters without verifying against each CLI's actual Unicode output. Braille Unicode sub-ranges look similar but different TUI frameworks use entirely different code points. Agent CLIs also silently change output formats between versions.

## Lesson

When configuring agent detection patterns, source Unicode characters directly from the CLI binary/source — never copy from another agent's config. Braille spinner sets from different TUI frameworks (Bubble Tea, Ink, ora) use different code points entirely. Whitespace in regex between symbol and text should use `\s*` not `\s+` unless the whitespace is guaranteed. Build test coverage for every agent verb and spinner frame before shipping.

## Applies when

Adding or updating detection patterns in `shared/config/agentRegistry.ts` for any agent. Especially when an agent reports as perpetually "working" or never transitions to "idle" correctly.
