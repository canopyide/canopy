## Context

Terminal panels are xterm.js instances that get reparented between DOM containers when focus changes (agent dock vs. grid layout). The `attach()` method appends the terminal element to a new container, then schedules a `terminal.refresh()` inside a `requestAnimationFrame`.

## Problem

Clicking an agent terminal to focus it caused a brief visual flash (~1 frame) in ~70-80% of cases. The terminal content momentarily went blank before rendering correctly.

## Root cause

The DOM element was made visible immediately on `appendChild`, but the xterm content was not refreshed until the next `requestAnimationFrame`. This created a one-frame window where the terminal container was visible but xterm had not yet repainted — producing a blank/white frame.

The issue was specific to the reparent path (agent dock ↔ grid transitions), not to terminals that stay in the same container.

## Lesson

**When reparenting a canvas-rendered element (xterm, WebGL), hide it until the renderer has repainted.** Set `opacity: 0` immediately before `appendChild`, then restore `opacity: 1` inside the `requestAnimationFrame` callback _after_ `terminal.refresh()` completes.

```ts
element.style.opacity = "0";
container.appendChild(element);
requestAnimationFrame(() => {
  terminal.refresh(0, rows - 1);
  element.style.opacity = "1";
});
```

**Use a safety timeout** (`REPARENT_OPACITY_TIMEOUT_MS = 200ms`) to restore visibility even if the `rAF` callback never fires (e.g., if the tab is backgrounded or rAF is throttled). Without it, the terminal could remain permanently invisible.

**Reset opacity on early-return paths** (e.g., when the target element is null). Otherwise a prior `opacity: 0` can persist if the attach is interrupted.

Using `visibility: hidden` instead of `opacity: 0` would also work but causes layout shifts; `opacity: 0` preserves layout.

## Applies when

- Reparenting xterm.js terminal elements between DOM containers
- Any canvas or WebGL element moved in the DOM that needs a repaint before display
- WebContentsView or offscreen rendering elements that need post-attach refresh
