## Context

The `AgentStateMachine` polls terminal output every 50ms and uses a `WorkingSignalDebouncer` to require sustained activity before transitioning from "waiting" back to "working". The debounce threshold was 300ms.

## Problem

When a user switches worktrees or resizes panels, the renderer re-lays out terminal panels. This causes brief bursts of visible-line changes detected by the polling cycle (not necessarily PTY resize events). These transients last 100–300ms — enough to exceed the 300ms debounce threshold — producing false waiting→working transitions. The agent state indicator would flash "working" and then settle back to "waiting" after the 4-second idle debounce.

## Root cause

Two distinct mechanisms exist to suppress false transitions: (1) `notifyResize()` suppresses activity for 1000ms after a PTY resize, and (2) `workingRecoveryDelayMs` requires sustained signal before a transition. However, worktree switches cause visible-line changes that don't always coincide with PTY resize events — they come from renderer reflows. Since 300ms is within the duration of typical layout shifts, the debounce was ineffective.

Real LLM agent output is sustained for multiple seconds to minutes. A debounce of 1500ms eliminates false positives from layout shifts while having zero impact on detecting genuine agent work.

## Lesson

When debouncing transitions triggered by output heuristics (polling-based), the debounce window must be longer than the longest expected transient from UI state changes, not just longer than network jitter. Layout reflows can produce 100–300ms of spurious signals. For agent-state detection, 300ms is far too short — 1500ms is the minimum safe value to avoid false transitions from worktree switches and panel resizes.

## Applies when

- Tuning `workingRecoveryDelayMs` or any debounce threshold in `ActivityMonitor`
- Investigating agent state flicker (rapid working↔waiting flips)
- Adding new layout operations that might affect terminal content visibility
- Designing polling-based state machines that observe terminal output
