## Context

A `TwoPaneSplitLayout` had a `useEffect` with a comment stating it should only run on mount and unmount. Its deps were `[terminals, commitRatioIfChanged]`. The `terminals` prop came from `ContentGrid` as an inline `.slice().map().filter()` expression, producing a new array reference on every parent render.

## Problem

Adding a third panel while in two-pane split mode corrupted the grid layout. The effect's cleanup was firing on every parent re-render (because `terminals` was a new reference each time), issuing spurious `lockResize` calls and committing stale ratios mid-transition.

## Root cause

React compares dep entries by reference. A new array literal from the parent is `!==` the previous one, so any effect that depends on it runs cleanup + setup every render. A comment saying "should only run on mount/unmount" is not enforcement — the deps array is the enforcement. This is especially pernicious when the effect's cleanup has side effects (not just listener removal) like committing persisted state.

## Lesson

If an effect is conceptually "mount/unmount only", its deps MUST be `[]`. Any other deps array makes it a reactive effect. Reference-unstable props (inline arrays, objects, lambdas from parents) turn every render into a cleanup cycle — audit cleanups that perform writes or external commits, not just listener removal. Prefer reading dependencies via ref/selector inside the effect body when the effect should not re-run, rather than listing them in deps.

Watch for parents that construct arrays via `.slice().map().filter()` inline — these are invisible render churners. Tooling like `eslint-plugin-react-hooks` flags missing deps but does NOT flag "deps too broad for this effect's intent".

## Applies when

Writing a `useEffect` whose cleanup has side effects beyond listener removal. Receiving arrays/objects as props that are constructed inline in the parent. Debugging mysterious state commits or layout thrash during unrelated re-renders. Reviewing any effect whose comment claims it only runs on mount.
