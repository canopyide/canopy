## Context

`ActivityMonitor` drove agent state transitions (working / waiting / idle) via output pattern analysis. It had a known weakness: agents doing silent CPU-intensive work (large diffs, API calls with no streaming output) would time out to "idle" prematurely after 3 minutes of silence.

## Problem

Output-based heuristics have no signal during silent processing phases. A 3-minute silence timeout is too long for responsive UX but too short for agents doing heavy background work.

## Root cause

The `ProcessStateValidator` interface only exposed `hasActiveChildren()` (child process count). CPU utilisation from the already-available `ProcessTreeCache` was not wired into the state machine.

## Lesson

**CPU as a supplementary, not primary signal:** output patterns remain authoritative for nuanced states (waiting-for-approval vs waiting-for-input). CPU is a coarse tiebreaker that prevents premature timeouts.

**Four-quadrant model:**

- High CPU + output → high-confidence "working"
- High CPU + no output → "working" (silent processing) — block `CompletionTimer` from firing
- Near-zero CPU + no output → faster transition to "waiting" or "idle"
- Near-zero CPU + output → "working" (API-streamed response, server does the work)

**Threshold (10%):** macOS %cpu is a decaying average across all cores; 10% flags meaningful work without triggering on idle background noise. The value is exported as `HIGH_CPU_THRESHOLD_PERCENT` for use across the pipeline.

**Graceful degradation:** wrap the CPU signal in a try/catch or optional check — if `isCpuBusy()` is unavailable (e.g. in unit tests or on unsupported platforms), fall back to output-only behaviour without throwing.

**Polling lag awareness:** macOS CPU values have 30–60s decay lag. CPU may report high values briefly after an agent stops. Existing `WorkingSignalDebouncer` timers absorb this without extra logic.

## Applies when

- Tuning agent idle-detection timeouts
- Extending `ProcessStateValidator` with new process-level signals
- Any state machine that relies on output silence to determine completion
