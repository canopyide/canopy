## Context

Hovering anywhere on a worktree card caused all file names in the file list to turn gray, even files not being hovered.

## Problem

Tailwind's `group` and `group-hover:` classes work on the nearest ancestor with `group`. When both a card-level container and a file-row container both used the plain `group` class, the file-row's `group-hover:` styles were also triggered by the card-level hover, affecting all rows simultaneously.

## Root cause

Unnamed Tailwind groups (plain `group`) are shared — any `group-hover:` descendant responds to hover on **any** ancestor with `group`. When the card container has `group` and file rows also have `group`, the file row's hover selectors fire on card-level hover too.

## Lesson

Use Tailwind's named groups (`group/card`, `group/file-row`) when hover scoping needs to be independent at multiple levels. Name the group on the container and reference it explicitly with `group-hover/card:` or `group-hover/file-row:`. Plain `group` should only be used when there is exactly one hover scope in the component tree.

## Applies when

- Components with nested interactive elements that need independent hover states
- Worktree cards, list items, or any card-with-rows pattern
- Tailwind v4 — named groups have been available since v3.2
