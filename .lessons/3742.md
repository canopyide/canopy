## Context

Git's `core.fsmonitor` config option specifies a command executed on every working tree query — including read-only operations like `git status`. The simple-git library is a thin wrapper around the `git` binary and inherits all local `.git/config` settings. IPC handlers accepting `cwd` from the renderer were calling `simpleGit(cwd)` without overriding any dangerous config options, and `validateCwd()` only checked for non-empty strings, not absolute paths.

## Problem

A malicious `.git/config` containing `core.fsmonitor = malicious-command` would execute that command whenever any git IPC handler ran, including status and diff operations. Multiple CVEs exist for this attack surface in simple-git (CVE-2022-25912, CVE-2022-24066, CVE-2022-24433).

## Root Cause

Two gaps existed simultaneously:

1. `validateCwd()` accepted relative paths (e.g., `../malicious-repo`) — an attacker-controlled renderer could direct git operations to arbitrary directories
2. All `simpleGit(cwd)` calls in IPC handlers trusted the target directory's `.git/config` entirely — no `-c` overrides were applied

The workspace host process already set `GIT_TERMINAL_PROMPT=0`, but this pattern was not applied to main process IPC handlers.

## Lesson

Create a `createHardenedGit(cwd)` wrapper that passes the following `-c` flags on every invocation via the `config` array in the `simpleGit()` constructor options:

```typescript
const HARDENED_GIT_CONFIG = ["core.fsmonitor=false", "core.pager=cat", "protocol.ext.allow=never"];
```

- `core.fsmonitor=false` — disables arbitrary command execution on status/diff
- `core.pager=cat` — prevents pager invocation that could block the process
- `protocol.ext.allow=never` — blocks the dangerous `ext::` transport (does NOT affect ssh/https)

Pair this with enforcing `path.isAbsolute(cwd)` in all `validateCwd()` calls. Relative paths from the renderer should always be rejected.

The `simpleGit` constructor's `config` option uses `-c key=value` which overrides repo-local `.git/config` — this is the correct layer to enforce restrictions.

## Applies when

- Any new IPC handler that calls `simpleGit(cwd)` with renderer-supplied paths
- Reviewing git operation code in `electron/ipc/handlers/`
- Accepting any path input via IPC that will be used as a git working directory
