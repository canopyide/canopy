## Context

The browser panel has a screenshot feature that captures the webview and copies the PNG to the clipboard. It used the Web Clipboard API (`navigator.clipboard.write()` with a `ClipboardItem`).

## Problem

On Linux (both X11 and Wayland), writing binary PNG data via `navigator.clipboard.write()` crashed the entire renderer process in Electron 41. The crash bypassed all JS error handling — try/catch could not catch it.

## Root cause

`navigator.clipboard.write()` with binary `ClipboardItem` data is unstable in Electron 41's renderer on Linux. The crash is at the native level, making it unhandleable in JavaScript.

## Lesson

Never use `navigator.clipboard.write()` with binary/image data in Electron renderer processes — route all clipboard image writes through IPC to the main process, which uses `electron.clipboard.writeImage()` via `nativeImage.createFromBuffer()`. This is stable on all platforms (macOS, Windows, Linux). For clipboard text, `navigator.clipboard.writeText()` is generally safe, but for binary data always use the main-process path.

## Applies when

Any feature that writes images or binary data to the clipboard in an Electron renderer or webview. Screenshots, copy-as-image, drag-out-to-clipboard.
