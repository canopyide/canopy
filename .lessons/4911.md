## Context

On project open, `stateHydration` restores persisted panel state from the previous session. Exited agent terminals (PTY process gone, exit code 0) should not be restored as live panels.

## Problem

Exited agent terminals reappeared as blank phantom panels on every project open.

## Root cause

The orphan filter in `stateHydration` only excluded `default-` prefixed terminals when no saved panels existed. It did not exclude agent terminals whose `hasPty` was `false`/`undefined` and exit code was 0. A secondary restoration path also lacked this guard, so exited agents slipped through both filters and got reconnected as if they were live terminals.

## Lesson

When filtering persisted terminals during hydration, apply agent-type checks at every restoration path independently — the reconnect manager applying a guard downstream is not sufficient if earlier paths also restore state. The correct check is: if `hasPty` is `false` or `undefined` AND the panel kind is an agent type AND exit code is 0 (or absent), skip restoration entirely. Don't rely on a single entry-point guard; enumerate all code paths that restore terminals.

## Applies when

Any change to `stateHydration`, panel persistence, or session restore logic. Adding a new panel kind that has a lifecycle (can exit or complete).
