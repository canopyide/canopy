## Context

`ReviewHub` rendered a commit message textarea. The worktree dashboard polls git status on a regular interval, causing the parent component to re-render. The textarea was rendered inline within the component tree that was affected by these re-renders.

## Problem

The textarea lost focus during typing and during idle periods. The user would be mid-keystroke and suddenly be unable to type. This happened because git status polling triggered state updates in the parent, which caused `ReviewHub` to unmount and remount its textarea DOM element, destroying browser focus.

## Root cause

The `commitMessage` state lived inside `ReviewHub`. Each time git status polling triggered a parent re-render, the parent's changed props caused `ReviewHub` to re-render. If React's reconciliation decided to recreate the component subtree (due to component identity changes or key instability), the textarea DOM element was replaced, destroying focus. Additionally, `onChange` on the textarea updated local state that triggered further re-renders.

Fix: lift `commitMessage` state up to the parent (`WorktreeCard`/`WorktreeOverviewModal`) and pass it as a controlled prop; use `useRef` for the value inside the textarea's `onChange` to avoid unnecessary state-triggered re-renders; memoize the commit form section.

## Lesson

In React, any component that contains a focused input must be stable across re-renders triggered by unrelated state updates. If a parent component re-renders due to polling (git status, resource metrics, etc.), child components containing inputs can lose focus if their identity changes. Strategies:

- Lift input state to the re-rendering parent (pass as controlled prop)
- Use `useRef` for the value to decouple `onChange` from render cycles
- Memoize the input-containing section with `React.memo` or `useMemo`

Never leave a textarea's state inside a component that gets re-rendered by polling-driven parent updates.

## Applies when

- Building any input/textarea inside a component tree that re-renders from polling
- Investigating focus loss bugs in worktree cards, review hub, or any git-status-driven UI
- Designing controlled inputs that live near frequently-updating store subscriptions
