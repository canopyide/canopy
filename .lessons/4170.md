## Context

When the worktree sidebar was shown or hidden, all visible terminals were resized as the grid reflowed. Codex CLI interprets a PTY `SIGWINCH` resize signal as a prompt to enter interactive mode, disrupting an in-progress agent run.

## Problem

The sidebar uses a CSS transition (smooth animation), so during the animation the terminal dimensions change continuously. Each intermediate size emitted a PTY resize to all running processes. Codex CLI specifically treats a resize signal as user intent to enter interactive mode.

## Root cause

`TerminalResizeController` was firing `proposeFit()` on every ResizeObserver callback during the CSS transition. There was no concept of "layout is transitioning — hold resize signals."

## Lesson

Any animated layout change (sidebar toggle, panel split, drag resize) that changes terminal container dimensions must gate PTY resize propagation. The fix pattern:

1. Set `sidebarTransitioning = true` on `TerminalInstanceService` at transition start
2. In `proposeFit()`, skip the resize IPC call while `sidebarTransitioning` is true
3. After `SIDEBAR_TRANSITION_MS`, flush a single resize with the final dimensions and clear the flag

This reduces N intermediate `SIGWINCH` signals to exactly 1 final signal. The `SIDEBAR_TRANSITION_MS` constant must be kept in sync with the CSS transition duration.

CLI tools that interpret `SIGWINCH` as interaction signals (Codex, some ncurses apps) are sensitive to spurious resizes. Any Electron app with animated layout transitions containing terminal panels needs this pattern.

## Applies when

Any layout animation that resizes terminal panels: sidebar toggle, panel split/join, window resize with transitions, responsive layout changes.
