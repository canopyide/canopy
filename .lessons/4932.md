## Context

Canopy reads PATH from the Windows registry (`HKCU\Environment` and `HKLM\SYSTEM\...`) to build the process environment for terminals and tool checks.

## Problem

All tool availability checks failed on Windows (showed ❌). The PATH set on `process.env` contained raw `%USERPROFILE%\AppData\...` tokens that were never expanded, making every path resolution fail.

## Root cause

`readWindowsRegistryPath()` was returning the raw registry string as-is. Windows registry values of type `REG_EXPAND_SZ` store paths with `%VAR%` environment variable references that the OS expands at read time — but the Node.js registry reading code was returning the literal string, not the expanded value.

## Lesson

When reading `REG_EXPAND_SZ` registry values on Windows, always expand `%VAR%` tokens before use. Implement an `expandWindowsEnvVars()` helper that resolves tokens against the current `process.env`. Also remember that common Node.js tool managers (npm global bin at `AppData\Roaming\npm`, Volta, pnpm, fnm, nvm-windows) are not in the system PATH by default and must be added manually to `applyWindowsExtraPaths()`.

## Applies when

Reading any PATH or directory value from the Windows registry. Adding support for a new Windows-specific tool manager path.
