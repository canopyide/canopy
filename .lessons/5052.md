## Context

The browser panel has a URL bar where users can navigate by typing a URL and pressing Enter. Under the hood, navigation calls `webview.loadURL()`.

## Problem

Pressing Enter on an already-loaded URL had no effect. The user expected the page to reload (matching standard browser behaviour), but nothing happened.

## Root cause

`webview.loadURL()` is a no-op when the URL being loaded is identical to the currently loaded URL. There is no error thrown — the call simply does nothing. This differs from what users expect, since every major browser reloads the page on Enter regardless of whether the URL changed.

## Lesson

When using Electron's `<webview>` or `WebContentsView`, `loadURL()` with the same URL does not reload the page. When implementing a URL bar where Enter should always reload, explicitly compare the submitted URL to the current URL and call `webview.reload()` instead of `loadURL()` when they match.

```typescript
if (submittedUrl === currentUrl) {
  webview.reload();
} else {
  webview.loadURL(submittedUrl);
}
```

## Applies when

Implementing browser URL bar navigation in Electron webview or WebContentsView components.
