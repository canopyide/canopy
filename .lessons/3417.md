## Context

OpenCode uses the Bubble Tea framework for its TUI, which requires alt-screen mode to render its full-screen interface. Canopy's PTY layer had `blockAltScreen: true` configured for OpenCode (matching the default for other agents), and also injected `CI=1` and `NONINTERACTIVE=1` environment variables that caused `termenv` to degrade to ASCII/no-color mode.

## Problem

OpenCode's TUI rendered as plain ASCII text with no UI chrome. The Bubble Tea framework detected the CI environment and switched to a degraded terminal mode, and the alt-screen block prevented the full-screen render from activating.

## Root cause

Two separate issues:

1. `blockAltScreen: true` prevented Bubble Tea's alt-screen switch (SMCUP sequence)
2. `CI=1` and `NONINTERACTIVE=1` are injected for agent agents to make them non-interactive, but `termenv` (Go's terminal environment detection library) uses these variables to disable color and TUI rendering

The agent exclusion list (`AGENT_ENV_EXCLUSIONS`) for CI env injection existed for Gemini but was not extended to OpenCode.

## Lesson

**Per-agent PTY configuration must match the agent's actual TUI requirements**. Agents using full-screen TUI frameworks (Bubble Tea, Ink, etc.) need:

- `blockAltScreen: false` in `agentRegistry`
- Exclusion from `AGENT_ENV_EXCLUSIONS` in `terminalShell.ts` to prevent CI/non-interactive env injection

When onboarding a new agent, check:

1. Does it use alt-screen? → `blockAltScreen: false`
2. Does its TUI detect CI env and degrade? → Add to `AGENT_ENV_EXCLUSIONS`

`CI=1` injection is appropriate for CLI tools that need to suppress prompts, but breaks TUI frameworks that use it as a rendering signal.

## Applies when

Adding any new agent that uses a full-screen TUI framework. Verify alt-screen and env injection settings explicitly rather than copying defaults from other agents.
