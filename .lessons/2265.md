## Context

Dev Preview parses the first URL a dev server prints on stdout (e.g. `Local: http://localhost:5173/`) and immediately points the Electron webview at it.

## Problem

Intermittent white-screen on panel startup. The webview fired `did-fail-load` with `ERR_CONNECTION_REFUSED` and never recovered.

## Root cause

Every major dev server — Vite, Next.js, Webpack Dev Server — prints its URL _before_ its HTTP listener is ready to accept connections. The gap is tens to hundreds of milliseconds and widens on cold starts. Pointing the webview at the URL the instant it appears in the terminal is a guaranteed race on fast machines with slow server startups.

## Lesson

After detecting a URL, poll the server with HEAD requests at ~500ms intervals up to a 30s timeout before transitioning to `running`. Accept 4xx as "ready" — some dev servers return 405 to HEAD. Cancel the poll on session stop, restart, terminal exit, or new URL detection. Attach an error handler to the polling promise to prevent unhandled-rejection crashes. When dead terminals are respawned, clear stale readiness state so the new session re-polls from scratch. Use a signal-identity guard (e.g. capture an `AbortController.signal` and compare on callback) so a stale poll completion cannot flip a newer session to running.

## Applies when

Wiring any component to a local dev server URL discovered from stdout, or debugging intermittent white-screen / `ERR_CONNECTION_REFUSED` on Dev Preview startup.
