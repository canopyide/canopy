## Context

Removing a project cleared the active selection by calling `store.set('projects.currentProjectId', undefined)`. This had been working on electron-store v10 and earlier.

## Problem

Deleting a project whose folder had been renamed or moved crashed the app. The crash traced to the `store.set(key, undefined)` call.

## Root cause

electron-store v11 introduced a breaking change: `store.set(key, undefined)` now throws instead of silently clearing the key. The documented way to clear a key is `store.delete(key)`. The old pattern is idiomatic in Conf-style stores and easy to overlook when upgrading — there is no deprecation warning, it goes straight to a thrown error at the call site.

## Lesson

On electron-store v11+ (currently `^11.0.2`), always use `store.delete(key)` to clear a key, never `store.set(key, undefined)`. When upgrading electron-store, grep the codebase for `\.set\([^)]*,\s*undefined\)` patterns before merging. The v11 changelog is worth re-reading when bumping — several APIs changed subtly, not just this one.

## Applies when

Writing or reviewing persistence code that clears stored values in the main process. Upgrading electron-store across a major version. Investigating crashes in code paths that nullify config entries. Any new handler that needs to "reset" a nested store key.
