## Context

AI agents run long tasks (code generation, test runs) that can take many minutes. If the system sleeps mid-task, the PTY process may pause or disconnect, breaking the agent's workflow.

## Problem

No mechanism existed to prevent system sleep while agents were actively working.

## Lesson

**Electron's `powerSaveBlocker` API requires explicit start/stop management** — `powerSaveBlocker.start('prevent-app-suspension')` returns an integer ID that must be passed to `powerSaveBlocker.stop(id)`. Losing this ID leaks the blocker permanently until process exit.

**Multi-agent tracking requires a counter, not a boolean.** With multiple concurrent agents, a single boolean "is any agent working?" is insufficient — you need to count active agents so that the blocker is only released when _all_ agents finish, not just the first one to complete.

**Safety timeout is mandatory.** If agent state tracking gets stuck (state machine never transitions to idle), the blocker would prevent sleep indefinitely. The implementation uses a 4-hour hard timeout as a leak prevention backstop — it logs a warning and releases the blocker.

**Subscribe to the event bus, not React state.** The PowerSaveBlocker is a main-process service; it reads agent state transitions from the event bus rather than polling the renderer store.

## Applies when

- Any Electron API that acquires a system resource by ID (blocker, wake lock, keyboard shortcut)
- Services that track "active count" of concurrent operations
- Any long-running operation where silent resource leaks have user-visible consequences
