## Context

`voiceCorrection.ts` builds AI prompts for speech-to-text correction that include the repository name extracted from the project path.

## Problem

On Windows, the full path (e.g. `C:\Users\deven\Desktop\canopy`) was being sent as the repository name in the prompt instead of just `canopy`. Voice correction quality degraded because the AI context was polluted with the full path string.

## Root cause

`projectPath.split("/")` only handles POSIX separators. Windows paths use `\`, so the entire path was treated as a single segment. The same bug existed in four renderer components that display directory names.

## Lesson

Never use `.split("/")` to extract path segments in code that runs on all platforms. Use `.split(/[/\\\\]/).filter(Boolean)` to handle both forward slashes (macOS/Linux), backslashes (Windows), and trailing separators. The `filter(Boolean)` also handles the empty string from a trailing slash.

```ts
// BAD
const repoName = projectPath.split("/").pop();

// GOOD
const parts = projectPath.split(/[/\\]/).filter(Boolean);
const repoName = parts[parts.length - 1];
```

## Applies when

- Any code in `shared/` that extracts a basename or last segment from a file path
- Renderer components displaying a directory or project name derived from a path
- Any place `path.basename()` would be the correct call but is not available (shared code runs in renderer without Node.js `path` module)
