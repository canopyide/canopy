## Context

The dock allows terminals to be minimised to a bar. Click expands them; double-click moves them back to the grid. Terminals track their xterm viewport visibility via `IntersectionObserver` and an `attachGeneration` counter.

## Problem

After a previous "fix" commit (`7b71ccb3f`), docking was broadly broken: single-click had a 250ms delay and sometimes failed to open the popover; double-click moved the terminal to the grid but rendered it blank; terminals disappeared from the dock after dock→expand→collapse cycles.

## Root cause

Commit `7b71ccb3f` replaced separate `onClick`/`onDoubleClick` handlers with a single `onClick` using `e.detail` and a `setTimeout(250ms)` to distinguish clicks. This introduced two bugs:

1. The 250ms timer added a visible delay to single-click and could fire after a double-click was already handled, causing double-fire.
2. The double-click branch called `moveTerminalToGrid()` without calling `closeDockTerminal()`. The comment claimed `moveTerminalToGrid` "atomically clears `activeDockTerminalId`" — it does not. Only the `terminalStore.ts` wrapper does, not the registry-level function called directly.

Additionally, `terminalsById` was missing from `useMemo` deps in `ContentDock` and `ContentGrid`, so tab groups didn't recompute when a terminal's location changed. A stale `IntersectionObserver` callback could also call `setVisible(false)` on a re-attached terminal, hiding it.

## Lesson

Never use `setTimeout`-based click/double-click arbitration in a UI with tight state transitions — it introduces timing races and can fire handlers after component state has already advanced. Keep `onClick` and `onDoubleClick` as separate handlers. When a store action must clear ancillary state (like `activeDockTerminalId`), only the store wrapper layer knows to do it — calling the registry-level function directly bypasses that cleanup. When writing `useMemo` for derived layout data, always include all store slices the derived value depends on in the deps array.

## Applies when

- Implementing click vs double-click disambiguation in panel/dock components
- Writing Zustand actions that must clear multiple state fields — always use the store wrapper, not the slice function
- Writing `useMemo`/`useCallback` with Zustand store subscriptions — include all accessed store fields in deps
