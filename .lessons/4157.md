## Context

Opening the Bulk Command Center caused a "Maximum update depth exceeded" crash in React.

## Problem

The `useRecipeStore` selector used `.filter()` which always returns a new array reference even when the contents haven't changed. React saw a new reference on every store subscription notification, re-rendered the component, triggering more notifications, causing infinite re-renders.

## Root cause

In Zustand 5, `useStore(selector)` uses strict reference equality by default. Any selector that creates a new object/array (`.filter()`, `.map()`, spread, etc.) will fail this check and cause a re-render on every store change — even if the actual data is identical.

## Lesson

Always wrap Zustand selectors that return derived arrays or objects with `useShallow` from `zustand/react/shallow`. This performs a shallow comparison instead of reference equality, preventing spurious re-renders. Applies to any `.filter()`, `.map()`, `Object.entries()`, or spread patterns in selectors.

```ts
// Bad — new array reference every call
const filtered = useRecipeStore((s) => s.recipes.filter((r) => r.active));

// Good — stable reference when contents haven't changed
const filtered = useRecipeStore(useShallow((s) => s.recipes.filter((r) => r.active)));
```

## Applies when

- Any Zustand selector that returns a derived array or object
- "Maximum update depth exceeded" errors in components using Zustand
- Zustand 5 specifically (behavior was more forgiving in earlier versions)
