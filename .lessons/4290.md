## Context

Canopy uses `better-sqlite3` (`canopy.db`) for persistent storage. SQLite in WAL mode accumulates a `.wal` file that grows without bound unless checkpointed.

## Problem

No backup strategy existed. A corrupted `canopy.db` had no recovery path. The WAL file could grow unboundedly, wasting disk space.

## Root cause

Operational gaps — no backup, no WAL checkpoint scheduling, no corruption detection at startup.

## Lesson

**`better-sqlite3` `.backup()` is the correct hot-backup API.** It uses SQLite's online backup API, which is safe to call while the database is open and in use. It copies atomically to a destination file without locking writes for the entire operation.

**WAL checkpoint must be run before `closeSharedDb()`**, not just periodically. If the WAL grows large and the process exits without a final checkpoint, SQLite still recovers (WAL is an append log), but the next open is slower. A final checkpoint on shutdown keeps the main DB file canonical.

**Corruption detection at startup must happen before any writes**. The sequence is: open DB → run `PRAGMA integrity_check` → if corrupt, quarantine (rename to `.corrupted-TIMESTAMP`) → attempt restore from `.bak` → if no backup, initialize fresh. This prevents the app from writing corrupt data on top of corrupt data.

**Integrity check on open is cheap** for small databases. `PRAGMA quick_check` is faster than `PRAGMA integrity_check` and catches most corruption for production use.

**Preserve corrupted files** rather than deleting them — rename with a timestamp suffix for forensic analysis. A separate cleanup job (PR #4288) removes these after 30 days.

The `DatabaseMaintenanceService` schedules periodic backup (every 15 minutes) and WAL checkpoint (every 10 minutes) via `setInterval`. Start after `app.ready`, stop in `shutdown.ts`.

## Applies when

- Any SQLite database used in Electron (better-sqlite3 or node-sqlite3)
- Adding resilience to persisted stores that cannot be easily re-derived
- WAL mode databases without explicit checkpoint scheduling
