## Context

`QuickRun` spawns a new terminal tab when the user selects a directory and presses Enter. The handler is async — it awaits the PTY spawn IPC call before completing.

## Problem

Rapid double-Enter keypresses, or a `cd` command completing and re-triggering the handler, could fire the spawn twice before the first async call finished. This created a duplicate terminal tab.

## Root cause

The handler had no guard against concurrent invocations. React synthetic events can fire faster than async IPC round-trips (tens of milliseconds), so two calls could both pass the "should spawn" check before either received its response.

## Lesson

Use a `useRef` reentrancy guard for async event handlers that must not run concurrently:

```ts
const isRunningRef = useRef(false);

async function handleRunItem() {
  if (isRunningRef.current) return;
  isRunningRef.current = true;
  try {
    await spawnTerminal(...);
  } finally {
    isRunningRef.current = false;
  }
}
```

`useRef` (not `useState`) is correct here because the guard update must be synchronous and must not trigger a re-render. `useState` would batch the update and leave a window where two calls can both read `false`.

Also: always set `type="button"` on buttons inside or adjacent to forms to prevent accidental form-submission events triggering the handler a second time.

## Applies when

- Async IPC calls are triggered by keyboard events (Enter key especially)
- Any handler that must not run concurrently (spawning, file writes, network calls)
- Buttons inside form elements without explicit `type`
