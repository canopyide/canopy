## Context

The worktree sidebar and overview modal render cards for every open worktree. With many worktrees, the browser performs layout, paint, and style calculations for all cards even when most are scrolled out of the viewport.

## Problem

Adding `content-visibility: auto` to sortable cards broke drag-and-drop behavior when using dnd-kit. dnd-kit applies CSS transforms and transitions to the dragged element, and `content-visibility: auto` interferes with these by skipping rendering of the element's subtree during the drag.

## Root cause

`content-visibility: auto` causes Chromium to skip rendering work for off-screen elements. However, it also creates a new stacking context and can interrupt the transform/position calculations that dnd-kit relies on during an active drag.

## Lesson

When applying `content-visibility: auto` to sortable/draggable elements, disable it during active drag operations:

```tsx
style={{
  contentVisibility: isDragging ? 'visible' : 'auto',
  containIntrinsicSize: 'auto 180px',
}}
```

The `containIntrinsicSize` with an `auto` prefix caches the last known rendered height, preventing scrollbar jitter when elements transition between rendered and skipped states. Use a reasonable estimated height (e.g. `auto 180px` for sidebar cards, `auto 240px` for modal cards).

## Applies when

Adding `content-visibility: auto` to any elements that participate in drag-and-drop interactions, or that have CSS transforms applied to them programmatically.
