## Context

Switching between projects in Canopy requires resetting 15+ Zustand stores then rehydrating them with the incoming project's saved state. This is implemented via `projectStore.switchProject()` which called `resetAllStoresForProjectSwitch()` followed by a rehydration hook.

## Problem

The two-phase reset-then-rehydrate pattern produced a visible intermediate state: all stores were cleared to defaults before the new project's data was loaded. The terminal registry being emptied to `[]` and then rebuilt caused a visible flash of empty terminal panels during project switches.

## Root cause

The reset and rehydration were two separate Zustand updates, meaning React would render the intermediate "empty" state between them. The render cycle that caught the empty terminal registry (`[]`) produced a flash before the new panels appeared.

## Lesson

When switching between large state blobs in Zustand, prefer an atomic swap over a two-phase clear+rehydrate. If you must perform cleanup operations, identify which stores genuinely need teardown logic (e.g. detaching PTY listeners, clearing caches) versus which ones can simply be overwritten. Only run teardown on the former group; skip the intermediate clear for the latter and let rehydration overwrite directly. This eliminates the empty intermediate render cycle.

The pattern: `atomicSwapForProjectSwitch(newState)` sets the terminal registry to the new value in a single update, bypassing the empty `[]` state entirely.

## Applies when

Any multi-store state transition where one phase empties data that another phase immediately repopulates — project switching, workspace resets, view eviction/restoration.
