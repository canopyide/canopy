## Context

Switching projects via "Open Directory" caused duplicate terminal tabs to appear, multiplying with each switch.

## Problem

The `useProjectSwitchRehydration` hook was registering a new `project-switched` event listener on every project switch without cleaning up the previous one. Additionally, `project-switched` was being emitted from multiple places: `appLifecycle.ts`, `menu.ts`, and `windowServices.ts` — causing the handler to fire multiple times per switch.

## Root cause

Two compounding issues: (1) the hook used a pattern that re-registered its listener on each invocation rather than once on mount, causing listener accumulation. (2) Multiple call sites emitted the same event independently, meaning even a single listener would process duplicate events. Neither issue alone caused the full severity — the combination was required.

## Lesson

When a React hook registers a native IPC/event-emitter listener, always register it **once on mount** and return a cleanup function — never re-register inside the handler or on state changes. Also designate a single canonical emit site for each event (in this case `windowServices.ts`) and remove all other redundant emissions.

## Applies when

- Hooks that call `ipcRenderer.on()` or `window.electron.*` event subscriptions
- Any IPC event emitted from multiple locations (lifecycle, menu, services)
- Debugging "duplicate panels/tabs appearing after navigation"
