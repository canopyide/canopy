## Context

During project switching, previously-attached ("warm") xterm terminals were reused and remounted into the CSS grid. Saved `cols`/`rows` dimensions were supposed to be applied before first paint to avoid visible layout shift.

## Problem

`setTargetSize()` stored the target dimensions in memory but did not resize the terminal immediately. The actual `terminal.resize()` call only happened inside `attach()`, which deferred it via `requestAnimationFrame`. The sequence was:

1. `addTerminal()` mounts component into CSS Grid — grid uses flexible `1fr` units, not saved dimensions
2. `setTargetSize()` stores `targetCols`/`targetRows` — **no resize yet**
3. `attach()` fires in `requestAnimationFrame` → calls `terminal.resize()`
4. Terminal snaps to saved size → grid reflows → **visible layout shift**

The `requestAnimationFrame` deferral inside `attach()` was originally there because WebGL context creation requires the terminal to be in the DOM — but that constraint doesn't apply when the terminal is already attached.

## Root cause

`setTargetSize()` was designed for initial setup (before attachment) and never had a fast-path for warm/already-attached terminals. The `attach()` path always deferred resize even when the terminal was already fully initialised.

## Lesson

When restoring or reusing a stateful component that tracks its own dimensions (like xterm.js), distinguish between **cold** (first attach) and **warm** (re-attach/re-mount) paths:

- Cold path: dimensions can be deferred until the DOM is ready
- Warm path: call `resize()` synchronously before the first paint, since the terminal's renderer is already initialised

Add a `resizeIfWarm()` helper that checks `terminal.element` attachment state and calls `terminal.resize()` immediately if already attached. Call it from `setTargetSize()` so that warm terminals always paint at the correct size on the first frame.

## Applies when

Restoring xterm instances across project switches, panel reorders, or any re-mount scenario where saved dimensions need to be applied without a visible resize flash.
