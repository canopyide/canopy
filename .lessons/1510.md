## Context

Agent terminals (Claude, Codex, Gemini) bypass the SharedArrayBuffer visual stream and fall back to IPC because their high-rate output starves the SAB renderer. The IPC fallback had no flow control, so a chatty agent could queue unbounded data in the main process and OOM the app.

## Problem

SAB already has backpressure via producer/consumer pointers. When we skipped SAB for agents, we lost that mechanism entirely and did not replace it. Main process heap grew until crash.

## Root cause

There are two transports for terminal output (SAB and IPC) and only one had flow control. Anything routed to IPC — not just agents but also suspended/backgrounded terminals, or terminals before the renderer attaches — inherits this problem.

## Lesson

Every PTY output transport must have bounded queuing with PTY pause/resume. node-pty supports `pause()`/`resume()`, so use them: pause at ~90% of a per-terminal byte cap, resume at ~50%, drop with a telemetry event if the hard cap is exceeded, and always force-resume after a safety timeout (5s) so a stuck consumer cannot permanently freeze the PTY. Track exact byte counts — approximations drift and defeat the cap. Clean up all per-terminal backpressure state on exit or the maps leak across reused terminal IDs.

## Applies when

Adding any new PTY output transport, any direct `terminal:data` IPC path, or any consumer that might stall (backgrounded tiers, detached renderers, SAB suspension). If you find yourself calling `event.sender.send` with PTY data, you need flow control.
