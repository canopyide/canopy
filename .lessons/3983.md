## Context

Agent terminals must stay pinned to the bottom of their output as the agent writes. A regression caused the viewport to intermittently snap to line 0 (the very top of the scrollback buffer) mid-stream, forcing users to manually scroll back down. The issue was marked as a re-emergence of a previously fixed bug.

## Problem

xterm.js emits `onScroll` events not only for user-initiated scrolls but also for programmatic buffer operations — including when the internal buffer writes new lines and the base viewport index (`baseY`) increments. The `onScroll` handler in `TerminalInstanceService` used this event to detect "user scrolled back" and set `isUserScrolledBack = true`. During active agent output, these programmatic scroll events incorrectly flagged the terminal as user-scrolled, which then blocked the auto-scroll pinning logic from working.

## Root cause

The `onScroll` handler couldn't distinguish user scroll events from scroll events triggered by xterm.js buffer operations during `writeToTerminal`. Adding an `_isWriting` guard flag — set to `true` around the `write()` call and `false` after — allows the `onScroll` handler to skip updating scroll state during writes. After the write completes, `scrollToBottomIfPinned()` is called to ensure the viewport stays at the bottom.

The `TerminalResizeController` also needed updating: it checks `isWriting` before overriding the scroll pin state during a resize event, to avoid racing with an in-progress write.

## Lesson

xterm.js `onScroll` fires on both user scroll and programmatic buffer writes. Any scroll state tracking logic that runs in `onScroll` must be guarded against writes. Use a write-guard flag (`_isWriting`) set around the write call, and explicitly re-pin after each write completes. The same guard must be checked in resize handlers that interact with scroll state.

## Applies when

- Implementing auto-scroll / scroll-pinning in xterm.js terminals
- Any code that updates `isUserScrolledBack` or similar state from `onScroll`
- Terminal resize logic that captures scroll position before resize
- High-frequency output streams (agent terminals, build output)
