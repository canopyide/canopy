## Context

Platform detection (`isMac()`, `isWindows()`, `isLinux()`) was scattered across 11 inline `navigator.platform` checks in 9 files.

## Problem

Two issues with inline `navigator.platform` checks:

1. **No SSR safety**: `navigator` is not available in all execution contexts (e.g., web workers, preload script, SSR). Unguarded access throws at runtime in these environments.
2. **Fragmented logic**: each callsite independently implements the same string match (`navigator.platform.includes('Mac')`, `navigator.platform.toLowerCase().includes('win')`, etc.) with subtle inconsistencies.

## Root cause

No shared utility existed; developers added inline checks ad hoc.

## Lesson

Centralize platform detection in a single utility (`src/lib/platform.ts`) with:

- Cached results (evaluate once, return cached value on subsequent calls)
- SSR-safe guards (`typeof navigator !== 'undefined'`)
- Consistent string matching across all platforms

```ts
let _isMac: boolean | null = null;
export function isMac(): boolean {
  if (_isMac === null) {
    _isMac = typeof navigator !== "undefined" && /mac/i.test(navigator.platform);
  }
  return _isMac;
}
```

When updating `detectOS()` in third-party adapter files, delegate to the shared utilities instead of rolling independent checks — this ensures consistent behavior and makes the platform utility the single source of truth.

Test module cache isolation: `vi.resetModules()` is needed in tests that mock `navigator.platform` because the cached value persists across test cases otherwise.

## Applies when

Any codebase with more than one `navigator.platform` check. Especially important in Electron where code may run in main, preload, renderer, and worker contexts with different global availability.
