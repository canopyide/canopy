## Context

The bulk worktree creation dialog shows a list of planned worktrees with their creation status. The `planned` array was derived via `useMemo` from `worktreeMap`.

## Problem

As each worktree was created and landed in `worktreeMap`, the `useMemo` recomputed. The recomputation logic treated worktrees already in `worktreeMap` as "skipped" (since they already exist). The render filter then removed them from the visible list, causing completed items to disappear from the progress view immediately after creation.

## Root cause

The `planned` memo's purpose before execution (compute the plan) and during execution (track progress) are fundamentally different. Before execution, items in `worktreeMap` should indeed be excluded (they already exist). During execution, items should be kept visible regardless of whether they appear in `worktreeMap`.

Using a single reactive `useMemo` for both phases meant the "already exists" check that was correct for pre-execution planning became destructive during execution.

## Fix

Snapshot `planned` into a `useRef` when execution starts. During and after the run, render from the snapshot (stable). Only use the live `useMemo` for the pre-execution preview.

```ts
const plannedSnapshotRef = useRef<PlannedWorktree[] | null>(null);

// At execution start:
plannedSnapshotRef.current = planned;

// In render:
const displayList = isExecuting || isComplete ? plannedSnapshotRef.current : planned;
```

## Lesson

When a derived value is used for both planning (pre-execution) and progress display (during execution), the computation logic for each phase may be mutually exclusive. Freezing the plan into a `useRef` at execution start and reading from the snapshot during execution is the right pattern — it prevents reactive recomputation from invalidating the display mid-run.

## Applies when

- Any progress dialog or multi-step wizard that derives its item list from a store that is mutated as the operation proceeds
- Bulk create/delete flows where the source collection changes as items are processed
